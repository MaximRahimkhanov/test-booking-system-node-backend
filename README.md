# Booking System API

REST API для системи бронювання зустрічей між клієнтами та бізнесами.

Система дозволяє:

- створювати клієнтів та бізнес-користувачів
- клієнтам бронювати час у бізнесів
- переглядати, редагувати та скасовувати записи
- захищає від дубль-записів на один і той самий час

## Tech Stack

- Node.js
- Express
- MongoDB + Mongoose
- REST API

## 4️⃣ Основні API endpoints (ВАЖЛИВО)

```md
## API Endpoints

### Users

- `GET /users` — отримати список користувачів
- `POST /users` — створити користувача
- `GET /users/:userId` — отримати користувача по ID
- `PATCH /users/:userId` — оновити дані користувача
- `DELETE /users/:iuserId` — видалити користувача

## Bookings

- POST /bookings — створення запису
- GET /bookings/my — отримання записів клієнта або бізнесу
- PATCH /bookings/:id — редагування часу запису
- DELETE /bookings/:id — скасування запису

> Примітка:
> Скасування запису реалізоване як зміна статусу на CANCELLED,
> а не фізичне видалення з бази даних.
> Це дозволяє зберігати історію бронювань.
```
